PASS 3 RUNBOOK
Goal: prove the stack works through Docker and the public gateway, not via direct service shortcuts.

Standard startup
1. docker compose up -d --build
2. Wait until these URLs are healthy:
   - http://localhost:8080/health
   - http://localhost:8080/health/auth
   - http://localhost:8080/health/file
   - http://localhost:8080/health/preview

Automated smoke
1. Run:
   python scripts/gateway_smoke.py
2. Optional alternate base URL:
   CFS_BASE_URL=http://localhost:8080 python scripts/gateway_smoke.py

What the smoke covers
- gateway health
- register
- /me
- verify-email request + consume
- create folder
- upload file
- search via backend
- download bytes integrity
- move to trash
- restore
- permanent delete
- forgot-password
- reset-password
- login with new password
- logout + refresh-token revocation

Frontend verification
1. Open http://localhost:8080
2. Sign up a new user
3. Complete the verify-email route opened by the returned action link
4. Open file manager and upload a text file
5. Search by unique filename fragment
6. Open preview and confirm browser-native rendering for text/image/pdf types
7. Move the file to trash, restore it, then delete permanently

If smoke fails
- Check docker compose logs for gateway/auth/file/frontend first
- Re-check docs:
  - http://localhost:8080/docs/auth
  - http://localhost:8080/docs/file
  - http://localhost:8080/docs/preview
- If only the automated smoke fails, compare the failing step with the corresponding endpoint in PASS_1_CONTRACT_MATRIX.txt
